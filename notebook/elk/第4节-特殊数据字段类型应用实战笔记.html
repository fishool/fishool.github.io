                                                                      # Join 父子

                                                                      • join 字段类型关键字
                                                                      • relations 关系父子绑定关键字
                                                                      • routing 路由, 保证父子数据在同一分片中.
                                                                      • 添加子文档时, 需要指定父文档的id, 以及父文档的routing值 Join 父子open in new window

                                                                      查询时:

                                                                      • parent_id 查询父文档
                                                                      • has_child 存在子文档
                                                                      • has_parent 存在父文档
                                                                      DELETE /foolish-test-0001
PUT /foolish-test-0001
{
  "mappings": {
    "properties": {
      "companyName": {
        "type": "keyword"
      },
      "join_father_child": {
        "type": "join",
        "relations": {
          "father": "child"
        }
      }
    }
  }
}
# 添加父数据
PUT /foolish-test-0001/_doc/1
{
  "companyName": "foolish_111",
  "join_father_child": {
    "name": "father"
  }
}
# 添加子数据
PUT /foolish-test-0001/_doc/2?routing=1
{
  "companyName": "foolish_222",
  "join_father_child": {
    "name": "child",
    "parent": "1"
  }
}
# 添加子数据
PUT /foolish-test-0001/_doc/3?routing=1
{
  "companyName": "foolish_3333",
  "join_father_child": {
    "name": "child",
    "parent": "1"
  }
}

# 查询父数据
GET /foolish-test-0001/_search
{
  "query": {
    "has_child": {
      "type": "child",
      "query": {
        "match_all": {}
      }
    }
  }
}

# 根据父节点ID查询
GET /foolish-test-0001/_search
{
  "query": {
    "parent_id": {
      "type": "child",
      "id": "1"
    }
  }
}

                                                                      # nested 嵌套键值对

                                                                      ES 支持数组类型, 但数组类型在查询时, 会将数组中的所有子元素都作为一个文档进行过滤匹配,
                                                                      这样会导致查询结果中, 出现一些预期之外的数据. (逻辑与 不生效等情况)

                                                                      限制参数

                                                                      • 嵌套字段内容字段数量限制 默认50
                                                                      • 嵌套字段对象文档数量数量限制 默认10000 嵌套字段open in new window
                                                                      DELETE /foolish-test-0001
PUT /foolish-test-0001
{
  "settings": {
    "index.mapping.nested_fields.limit": 50,
    "index.mapping.nested_objects.limit": 10000
  }
}

                                                                      # 案例分析

                                                                      • 假设一个用户, 有多个订单, 每个订单有多个商品, 每个商品有多个属性
                                                                      • 错误的案例, 运用 array类型 (object)
                                                                      DELETE /foolish-test-0001
PUT foolish-test-0001
{
  "mappings": {
    "properties": {
      "user_id": {
        "type": "keyword"
      },
      "order": {
        "type": "object",
        "properties": {
          "order_id": {
            "type": "keyword"
          },
          "goods": {
            "type": "object",
            "properties": {
              "goods_id": {
                "type": "keyword"
              },
              "goods_name": {
                "type": "keyword"
              }         
             }
            }
          }
        }
      }
    }
}
# 添加数据
PUT /foolish-test-0001/_doc/1
{
  "user_id": "foolish_111",
  "order": {
    "order_id": "order_111",
    "goods": [{
      "goods_id": "goods_111",
      "goods_name": "goods_name_111"
    },
    {
      "goods_id": "goods_222",
      "goods_name": "goods_name_222"
    }]
  }
}

# 查询
GET /foolish-test-0001/_search
{
  "query": {
    "bool": {
      "must": [
        {
          "match": {
            "order.goods.goods_id": "goods_111"
          }
        },
        {
          "match": {
            "order.goods.goods_name": "goods_name_111"
          }
        }
      ]
    }
  }
}

# 查询商品id为 111 且 name为 222 的数据 也会被查询出来
GET /foolish-test-0001/_search
{
  "query": {
    "bool": {
      "must": [
        {
          "match": {
            "order.goods.goods_id": "goods_111"
          }
        },
        {
          "match": {
            "order.goods.goods_name": "goods_name_222"
          }
        }
      ]
    }
  }
}
                                                                      • 正确的案例, 运用 nested类型
                                                                      • 在查询时, 将嵌套字段的过滤条件, 放在 nested对象中, 指定 path 和 query
                                                                      DELETE /foolish-test-0001
PUT foolish-test-0001
{
  "mappings": {
    "properties": {
      "user_id": {
        "type": "keyword"
      },
      "order": {
        "type": "nested",
        "properties": {
          "order_id": {
            "type": "keyword"
          },
          "goods": {
            "type": "nested",
            "properties": {
              "goods_id": {
                "type": "keyword"
              },
              "goods_name": {
                "type": "keyword"
              }         
             }
            }
          }
        }
      }
    }
}
# 添加数据
PUT /foolish-test-0001/_doc/1
{
  "user_id": "foolish_111",
  "order": {
    "order_id": "order_111",
    "goods": [{
      "goods_id": "goods_111",
      "goods_name": "goods_name_111"
    },
    {
      "goods_id": "goods_222",
      "goods_name": "goods_name_222"
    }]
  }
}

# 查询
{
  "query": {
    "bool": {
      "must": [
        {
          "nested": {
            "path": "order.goods",
            "query": {
              "bool": {

                  {

                    }
                  },

                    "match": {
                      "order.goods.goods_name": "goods_name_111"
                    }
                  }
                ]
              }
            }
          }
        }
      ]
    }
  }
}

                                                                      # fields 单值多字段

                                                                      ES 提供了丰富的数据类型 , 有很多特殊场景需求, 我们需要根据一个字段, 提供不同的检索需求, 在 ES 中, 我们可以通过 fields 关键字, 来实现这个需求, 原始数据只存储了一份,有效避免空间浪费 官方文档open in new window

                                                                      DELETE /foolish-test-0001
PUT foolish-test-0001
{
  "mappings": {
    "properties": {
      "user_id": {
        "type": "keyword"
      },
      "address": {
        "type": "text",
        "fields": {
          "keyword": {
            "type": "keyword"
          }
        }
      }
    }
  }
}

# 添加数据
PUT /foolish-test-0001/_doc/1
{
  "user_id": "foolish_111",
  "address": "guangdong shenzhen nanshanqu"
}

# text分词查询
GET /foolish-test-0001/_search
{
  "query": {
    "match": {
      "address": "shenzhen"
    }
  }
}

# keword查询
GET /foolish-test-0001/_search
{
  "query": {
    "wildcard": {
      "address.keyword": "*shen*"
    }
  }
}

                                                                      # copy_to 复制字段

                                                                      • copy_to 可以指定复制多个字段, 且按照顺序复制;
                                                                        copy 的字段, 会被索引, 但是不会被存储, 也就是说, copy 的字段, 只能用于检索, 不能用于返回结果 官方文档open in new window
                                                                      DELETE /foolish-test-0001
PUT foolish-test-0001
{
  "mappings": {
    "properties": {
      "user_id": {
        "type": "keyword"
      },
      "address": {
        "type": "text"
      },
      "province": {
        "type": "keyword",
        "copy_to": "address"
      },
      "city": {
        "type": "keyword",
        "copy_to": "address"
      },
      "area": {
         "type": "keyword",
        "copy_to": "address"
      }
    }
  }
}

# 添加数据
PUT /foolish-test-0001/_doc/1
{
  "user_id": "foolish_111",
  "province": "guangdong",
  "city": "shenzhen",
  "area": "nanshanqu"
}

# 查询
GET /foolish-test-0001/_search
{
  "query": {
    "match": {
      "address": "shenzhen"
    }
  }
}

                                                                      # histogram 数值直方图类型

                                                                      用于存储表示直方图的预聚合数值数据的字段
                                                                      histogram字段主要用于聚合。为了便于聚合访问，histogram字段数据存储为二进制文档值，而不是编制索引。

                                                                      • histogram 数值直方图类型关键字 histogram 数值直方图类型open in new window

                                                                      # Runtime 运行时类型

                                                                      • Runtime 运行时类型关键字
                                                                      • 7.11.x 版本开始支持
                                                                      • 可以不提前定义字段, 在查询时动态添加字段 , 不占用存储, 运行时字段可以在查询和聚合中使用
                                                                      • "fields": ["*"] 运行时字段不会出现在 _source 中, 可以在查询时使用 fields 参数来获取运行时字段 Runtime 运行时类型open in new window

                                                                      目前支持的类型(type)

                                                                      • boolean
                                                                      • date
                                                                      • double
                                                                      • geo_point
                                                                      • ip
                                                                      • keyword
                                                                      • long

                                                                      script.source 为脚本内容, 语言为 painless

                                                                      # 案例

                                                                      DELETE /foolish-test-0001
PUT foolish-test-0001
{
  "mappings": {
    "properties": {
      "user_id": {
        "type": "keyword"
      },
      "salary": {
        "type": "long"
      }
    },
    "runtime": {
      "salary_month": {
        "type": "long",
        "script": {
          "source": "emit(doc['salary'].value / 12)"
        }
      }
    }
  }
}

# 添加数据
PUT /foolish-test-0001/_doc/1
{
  "user_id": "foolish_111",
  "salary": 12000
}

PUT /foolish-test-0001/_doc/2
{
  "user_id": "foolish_222",
  "salary": 1200
}

# 查询
GET /foolish-test-0001/_search
{
  "fields": ["*"],
  "query": {
    "range": {
      "salary_month": {
        "gte": 800
      }
    }
  }
}
