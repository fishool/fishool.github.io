                                                                              # 设置Mapping

                                                                              es存储内容

                                                                              es原文档存储 => 索引 _source 属性 luncene 存储 => 字段 store 属性 字段检索索引存储 => 字段 index 属性 列式文档存储 => 字段 doc_value属性

                                                                              各存储方式都可以单独配置

                                                                              • source: 是否存储原始数据, 默认为true
                                                                              DELETE /foolish-test-0001
PUT /foolish-test-0001
{
    "mappings": {
        "_source": {
        "enabled": false
        },
        "properties": {
        "user": {
            "store": true,
            "type": "text"
        },
        "post_date": {
            "store": true,
            "type": "date"
        },
        "age": {
            "store": true,
            "type": "integer"
        },
        "city": {
            "store": true,
            "type": "keyword"
        }
        }
    }
}

POST /foolish-test-0001/_doc/1
{
    "user": "kimchy",
    "post_date": "2009-11-15T14:12:12",
    "age": 30,
    "city": "shanghai"
}

GET /foolish-test-0001/_search
{
    "stored_fields": ["user", "post_date", "age", "city"]
    "query": {
        "match_all": {}
    }
}

   }
}

                                                                              # dynamic 是否动态创建字段

                                                                              • true: 容许索引字段动态扩展
                                                                              • false: 允许字段动态扩展, 但是仅仅存储原始数据,不能被检索/聚合
                                                                              • strict : 不允许字段动态扩展, 会报错
                                                                              DELETE /foolish-test-0001
PUT /foolish-test-0001
{
  "mappings": {
    "dynamic": "strict",
    "properties": {
      "user": {
        "type": "text"
      },
      "post_date": {
        "type": "date"
      },
      "age": {
        "type": "integer"
      },
      "city": {
        "type": "keyword"
      },
      "province": {
        "type": "keyword"
      }
    }
  }
}

                                                                              # 字符类型

                                                                              字符类型是ES 最常用之一, 内部实现是借助于lucene的分词器, 通过分词器将字符串分词, 然后存储到倒排索引中, 从而实现字符串的检索

                                                                              • Text: 用于存储文本, 会被分词器分词, 会被索引, 会被存储 (应用在全文检索领域) 文档open in new window
                                                                              DELETE /foolish-test-0001
PUT /foolish-test-0001
{
  "mappings": {
    "properties": {
      "message": {
        "type": "text",
        "analyzer": "standard"
      }
    }
  }
}

PUT /foolish-test-0001/_doc/1
{
  "message": "this is a test, fool is foolish"
}

PUT /foolish-test-0001/_doc/2
{
  "message": "China is a big country"
}

# 检索
GET /foolish-test-0001/_search
{
  "query": {
    "match": {
      "message": "foolish"
    }
  }
}
                                                                              • Keyword: 用于存储关键字, 不会被分词器分词, 会被索引, 会被存储(应用在精确值检索领域) keyword 类型划分
                                                                              • keyword: 用于存储关键字, 不会被分词器分词, 会被索引, 会被存储(应用在精确值检索领域)
                                                                              • constant_keyword: 固定值类型, 不会存储, 需要设置默认固定值,便于查询与统计, 避免错误和性能问题
                                                                              • wildcard: 通配符类型, 采用ngram分词器, 支持模糊查询, 性能比keyword差一些, 但是模糊查询比keyword强大
                                                                              DELETE /foolish-test-0001
PUT /foolish-test-0001
{
  "mappings": {
    "properties": {
      "keyword_message": {
        "type": "keyword"
      },
      "constant_message": {
        "type": "constant_keyword",
        "value": "foolish"
      },
     "wildcard_message": {
        "type": "wildcard"
     }
    }
  }
}

PUT /foolish-test-0001/_doc/1
{
  "keyword_message": "keyword123",
  "wildcard_message": "wildcard123"
}

POST _analyze
{
  "analyzer": "keyword",
  "text": ["keyword123"]
}


POST _analyze
{
  "tokenizer": "ngram",
  "text": ["wildcard123"]
}

                                                                              # 浮点数值类型

                                                                              • double: 64位双精度浮点数
                                                                              • float: 32位单精度浮点数

                                                                              # 日期类型

                                                                              # Range类型

                                                                              Range类型, 用于存储范围数据,
                                                                              相比传统类型数据库, 性能优势明显, 内部使用BDK树实现, 有许多应用场景,
                                                                              例如: 衣服尺寸 ,设计时都是按照一个范围值设计的. 175-180, 180-185, 185-190

                                                                              文档open in new window

                                                                              • integer_range: 整数范围
                                                                              • float_range: 浮点数范围
                                                                              • long_range: 长整数范围
                                                                              • double_range: 双精度浮点数范围

                                                                              # flattened 单一化类型

                                                                              • 传统上, 创建对象类型, ES会自动创建对应的Mapping结构, 如果对象结构复杂, 会导致Mapping结构过于复杂,
                                                                                发生 Mapping explosion
                                                                              • flattened 单一化类型, 可以将对象类型的数据, 单一化存储, 也能使用和Object同样的查询方式 文档open in new window
                                                                              DELETE /bug_reports
PUT bug_reports
{
  "mappings": {
    "properties": {
      "title": {
        "type": "text"
      },
      "labels": {
        "type": "flattened"
      }
    }
  }
}

POST bug_reports/_doc/1
{

  "labels": {
    "priority": "urgent",
    "release": ["v1.2.5", "v1.3.0"],
    "timestamp": {
      "created": 1541458026,
      "closed": 1541457010
    }
  }

POST bug_reports/_search

  "query": {

                                                                              # 字段类型自动推断

                                                                              # 启用字段类型自动推断

                                                                              • ES 默认不启用字段类型自动推断;
                                                                              • 常规情况下, 可以自动推测字符串类型 ,数值类型 , 日期类型
                                                                              DELETE /foolish-test-0001
PUT /foolish-test-0001
{
  "mappings": {
    "numeric_detection": true,
    "date_detection": true,
    "dynamic_date_formats": ["yyyy-MM-dd HH:mm:ss", "yyyy-MM-dd"]
  }
}

                                                                              # 索引字段数限制

                                                                              • ES 默认索引字段数限制为1000个, 可以通过设置index.mapping.total_fields.limit来修改
                                                                              DELETE /foolish-test-0001
PUT /foolish-test-0001
{
  "settings": {
    "index.mapping.total_fields.limit": 2000
  }
}

                                                                              # 对象类型嵌套深度

                                                                              • ES 默认对象类型嵌套深度为20层, 可以通过设置index.mapping.nested_fields.limit来修改, 建议不要超过3层
                                                                              DELETE /foolish-test-0001
PUT /foolish-test-0001
{
  "settings": {
    "index.mapping.depth.limit": 20
  }
}

                                                                              字段类型变更 , 历史数据不会自动变更, 需要手动重建索引 . 新增字段, 会自动生效
